# Occupancy Benchmark Data

This folder contains the first checked-in benchmark dataset for the traditional occupancy baseline.

## File

- `occupancy_synth_v1.csv` — deterministic synthetic occupancy sequence generated by `../generate_dataset.py`

## Why a synthetic benchmark first?

This benchmark is meant to be a **reasonable bridge task**, not the final scientific claim.

It gives us:

- a stable in-repo dataset for quick experiments
- temporal ordering suitable for sequential train/test splits
- correlated but imperfect sensor cues (`light`, `co2`, `temperature`, `humidity`, `humidity_ratio`)
- a few off-hours anomalies so the task is not reducible to a single threshold

## Generative assumptions

The generator simulates a small office-like environment sampled every 15 minutes over 14 days.

- weekdays: occupied during morning and afternoon work blocks
- weekends: lighter occupancy during a short midday block
- `co2` rises with occupancy and lags over time
- `light` follows daylight plus occupancy-driven indoor lighting, with occasional after-hours cleaning activity
- `temperature` rises modestly with occupancy and event spikes
- `humidity` shifts with time of day and occupancy
- `humidity_ratio` is derived from temperature/humidity with a small occupancy-sensitive perturbation

## Intended use

Use this dataset to make the traditional baseline reproducible while we carefully define how the same task should be mapped into REAL.

Example:

```bash
python 'Phase 8/occupancy_baseline/run_baseline.py' \
  --csv 'Phase 8/occupancy_baseline/data/occupancy_synth_v1.csv' \
  --window-size 5 \
  --epochs 60 \
  --output-json 'Phase 8/tests_tmp/occupancy_synth_v1_result.json'
```
