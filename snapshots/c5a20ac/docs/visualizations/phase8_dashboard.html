Design Constraints

Local-Only Learning

No global loss or gradient update path. Node observations expose only local state plus direct neighbors. ATP is tracked per-node and actions disappear when a node cannot afford them.

Metabolic Constraint

Edge investment lowers future route cost. Feedback returns upstream sequentially, not as a global broadcast. Action costs are metabolic and local — scarcity is the teacher.

Carryover & Transfer

Cross-session warm starts rehydrate node substrate and episodic survivors. Transfer between tasks is directional: Task A→B is robustly positive; B→A remains mixed.

Adaptive Admission

Source admission learns boundary openness from local success and friction. Efficiency is scored on metabolic cost vs returned feedback energy, not just throughput.

Context Sensitivity

Packets carry context bits and task IDs. Transform credit is context-bound, reducing warm-start interference across task contexts. Substrate remembers which actions worked where.

Morphogenesis

Nodes can bud new edges and child nodes when metabolic surplus exceeds threshold. Stale dynamic edges and isolated nodes are pruned or undergo apoptosis.

Why Asymmetry?
A → B succeeds because Task B uses a similar parity context signal. The branch-plus-transform credit path ties the preferred branch to the correct transform family. Context-bound credit from A's training seeds the right bias for B's patterns without needing retraining.
B → A fails because Task B's context_1 transform habit conflicts with Task A's expected transform. Contradiction debt is now gated by prior local commitment, which prevents over-damping, but the odd-context branch remains behind substrate-only carryover.
CVT-1 Computational Verification Tasks
CVT-1 is the first runnable computational scenario. Packets carry 4-bit input vectors; the sink scores task packets against a target transformation of the input. Context bits are set by parity of the previous packet's bits. The substrate must learn which transform to apply in each context.

CVT-1 Task Definitions
Signal format:
Input: 4-bit vector · Context: parity(prev bits) · Task ID: task_a/b/c
18-packet schedule:
Values 0b0001 → 0b0110 → 0b1011 → … scheduled across cycles. Context bit tracks sequential state from packet to packet.
Feedback:
Routing packets → legacy full pulse.
Task packets → feedback scaled by bit match quality.
Exact match = full reward, partial match = graded reward.

Credit / Debt System
Transform Credit
Returned when a transform leads to a good match. Stored per-transform, per-context, per-branch, and per branch+context combination.
Transform Debt
Accumulated when contradiction is detected (prior local commitment required to gate debt). Reduces maintenance priority for debt-heavy supports.
Branch-Context Credit
Ties a specific branch selection to the correct transform in a specific context. Most precise signal in the system.
Substrate Maintenance
Explicit maintenance lets nodes actively spend ATP to preserve high-value substrate entries rather than relying on slow natural decay.
Recently maintained entries are distinguished from support that is merely lingering — summaries expose this ratio.
Promoted context-action support can survive substrate-only carryover, inheriting part of the learned task structure even without episodic memory.

Carryover Paths
Full Carryover
Substrate + episodic memory + context-transform credit + branch-context structures. Maximum transfer but highest risk of interference from contradictory prior context.
Substrate-Only Carryover
Promoted context-action support + routing scaffold. Episodic layer clears, removing most interference but retaining structural routing habits. Intermediate transfer quality.
Cold Start
No prior memory. Baseline exploration. Used as the reference for all delta calculations in carryover and transfer comparisons.
Morphogenesis
Dynamic topology change governed by local metabolic energy. Nodes with surplus ATP and positive net energy can bud new edges or child nodes. Stale or negative-value dynamic edges are pruned. Isolated nodes undergo apoptosis. Topology is a metabolic question, not a design decision.

Transform Seeding on Growth
When a new node or edge is budded, the growth path is seeded with transform support from the parent node's existing credit/debt state. The top-2 generic transforms and top-2 context-specific transforms with positive net score (credit − 0.35×debt) are inherited. This lets learned task structure propagate into newly grown topology without requiring a fresh bootstrap period.
Architecture
The phase8/ package is organized around local-only substrate mechanics. All modules interact through the NativeSubstrateSystem coordinator.
node_agent.py
Wraps RealCoreEngine for each network node. Manages per-cycle perceive→select→execute→score→consolidate loop. Handles transform selection and local credit/debt updates.
substrate.py
Edge-scoped slow memory with support levels, transform credit/debt, context-transform credit, branch-context credit. Configurable decay, bistable threshold, velocity tracking.
environment.py
Small routing environment with per-node inboxes, packet expiry (TTL), sequential feedback flow, topology sync, and morphogenesis proposal collection.
admission.py
Source-boundary admission substrate. Learns boundary openness from local success and friction. Tracks metabolic efficiency (action cost vs returned feedback). Adaptive or fixed-rate modes.
topology.py
NodeSpec, EdgeSpec, TopologyState, TopologyManager. Governs morphogenesis: bud_edge, bud_node, prune_edge, apoptosis. Energy-governed growth checkpoints.
consolidation.py
Route-history promotion into maintained substrate. Promotes context-action supports to durable substrate. Handles maintenance priority based on credit/debt balance.
selector.py
Phase 8-specific local route selector. Combines edge support, transform credit, context bias, branch-context credit, and packet-aging pressure. Downweights stale episodic bias.
adapters.py
Observation, action, coherence, and memory bindings connecting the REAL Core Engine interface to Phase 8 packet routing mechanics.
models.py
Data classes: SignalPacket, FeedbackPulse, NodeRuntimeState, SignalSpec. Packets carry payload bits, context bits, task IDs, transform traces, and match quality scores.
scenarios.py
Reusable ScenarioSpec catalog: branch_pressure, sustained_pressure, detour_resilience, basic_demo, cvt1_task_a_stage1, cvt1_task_b_stage1, cvt1_task_c_stage1.
compare_cold_warm.py
Multi-seed cold vs warm full vs warm substrate comparison runner. Aggregates delivery, match quality, latency, ATP, admission, and transform diagnostics across seeds.
compare_task_transfer.py
Task A→B transfer harness. Trains on Task A, saves carryover, then tests each condition on Task B. Reports criterion window, examples-to-criterion, context-level diagnostics.
