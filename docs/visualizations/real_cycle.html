How a node actually runs

Every tick, each node executes the same anticipation-aware loop: observe -> recognize -> predict -> select -> execute -> score -> compare -> consolidate. This is RealCoreEngine.run_cycle(). There's no global loss, no backprop -- just local observation, local anticipation, local action, and feedback that writes into local substrate. Reading local state

The node calls observe_local() on the routing environment. This returns a flat dict of floats describing everything the node can see right now â€” its ATP level, inbox contents, downstream progress, and recent feedback.

Then modulate_observation() runs. It injects substrate support values as additional keys, and adds Gaussian noise to progress/congestion readings â€” more noise on edges with low substrate support. A well-learned connection is perceived more clearly.

Why noise? It models epistemic uncertainty at the edge level. If you haven't routed through a neighbor much, you have a fuzzier read on what's going on over there. High substrate support â†’ high clarity â†’ lower noise scale.

Building and filtering the action set

First, available_actions() builds the full menu: rest, one route:N per neighbor, four route_transform:N:T variants per neighbor, plus invest:N and maintain_edges when the inbox is empty and ATP allows.

Each route action has a use_cost â€” base 0.05 ATP, reduced by edge support and action support. If a neighbor has been heavily used and well-maintained, routing there is cheap. The node can only afford actions whose cost fits its current ATP budget.

The Phase8 selector then picks from affordable actions using substrate bias, memory, and a CFAR-style exploration/exploitation balance. If the selector is uncertain, it explores; if a route has strong substrate support, it exploits.

Firing the action

The chosen action is dispatched. route:N moves a packet from this node's inbox to neighbor N's inbox, charging the use_cost in ATP. route_transform:N:T does the same but applies transform T to the packet's bit payload first.

rest recovers ATP (partial regen). invest:N charges write_cost and boosts the edge's slow support value by +0.20. maintain_edges calls maintain_supports(), paying maintain_cost per active edge/action to reset their decay clock.

The environment observes state again after execution â€” this "after" snapshot is what the coherence model scores.

Substrate tick and pattern promotion

Every cycle the substrate ticks: all slow values decay by 0.025. If a value drops below the bistable threshold (0.25), decay accelerates by 2.5Ã—. Connections that aren't maintained will fade â€” fast.

When the node takes a rest action and episodic memory has â‰¥8 entries, consolidation runs. It scans recent route history: if a neighbor was routed to â‰¥2 times with positive mean delta, it seeds edge support (0.32) and builds a ConstraintPattern. Negative-delta routes decay instead.

This is learning writing into structure. Durable behavior shows up as nonzero slow values that survive decay â€” not as parameter vectors.

Bistable cliff: at support 0.25, a connection is either maintained and stable, or rapidly forgotten. There's no gradual middle ground â€” this creates discrete "learned vs. unlearned" edge states.
